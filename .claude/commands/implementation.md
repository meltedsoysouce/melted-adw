# TASK

## ARGUMENTS

- plan : 実装計画書のパス
  - markdown形式で出力されている
  - フェーズ分けがされており、これを順次実装していく

# FLOWS

以下の流れに即してタスクを遂行すること

## 1. 完了条件の確認

今回の開発タスクの完了条件を確認する

## 2. フェーズの依存関係を確認

各実装フェーズの依存関係を確認する。
このとき、並列実装で問題ない場合には、並列で実装する

## 3. 実装の実施

計画書に基づいて、実装を行うこと。

IMPORTANT:
あなたは全体の指揮をとること。実際の実装は、サブエージェントに実施させること。

## 4. 完了条件が達成できているか確認

修正が必要であれば、ユーザーにフィードバックすること
